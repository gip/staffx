# Deployment and Environment Model

## Required environment variables
- API core: DATABASE_URL, PORT.
- Auth: AUTH0_DOMAIN, AUTH0_AUDIENCE.
- Integration encryption: INTEGRATION_ENCRYPTION_KEY.
- Integration providers: NOTION_CLIENT_ID, NOTION_CLIENT_SECRET, GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET.
- Optional callback tuning: INTEGRATION_CALLBACK_ORIGIN.
- Assistant controls: STAFFX_ENABLE_CLAUDE_AGENT, STAFFX_AGENT_RUNNER_ID, STAFFX_PROJECTS_ROOT, and polling or timeout overrides.
- Web and desktop client build variables: VITE_API_URL, VITE_AUTH0_DOMAIN, VITE_AUTH0_CLIENT_ID, VITE_AUTH0_AUDIENCE.

## Process deployment topology
- API process runs as HTTP service reachable by web and desktop clients.
- PostgreSQL runs as durable backend store.
- Web client is served as browser application bundle.
- Desktop client is packaged Electron application.

## Migration and bootstrap steps
- Install workspace dependencies with pnpm.
- Run API migration before service start.
- Start API and clients in development through root scripts.
- Configure OAuth callbacks for both web and desktop flows.
