Web client uses BrowserRouter routing, Auth0 access-token acquisition, and typed transformations of v1 API payloads to shared UI contracts. It calls /v1 endpoints for all user-visible workflows except desktop-only direct local execution. It implements SSE event parsing with cursor tracking and resilient fallback polling.
